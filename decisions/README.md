# Architectural Decisions (ADR)

This folder contains the history of the project's soul — its architecture. We use the **ADR (Architectural Decision Record)** format to document significant changes.

## Standard ADR Template

Every new decision file should follow this structure:

- **Title:** `ADR-[000]: [Short Title]`
- **Status:** [Proposed / Accepted / Superseded by ADR-XXX]
- **Context:** What technical or business problem are we facing? What are the constraints?
- **Considered Options:**
    1. **Option A:** Pros and Cons.
    2. **Option B:** Pros and Cons.
- **Decision:** Which option was chosen and why?
- **Consequences:** What do we gain? What technical debt or complexity are we introducing?
